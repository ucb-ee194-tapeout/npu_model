# NPU Compiler Scaffold

This subtree contains a standalone MLIR compiler scaffold for the NPU model.

## Layering

- Public layer 1: `npu_kernel` dialect (kernel semantics)
- Public layer 2: `npu_schedule` dialect (scheduled pipeline semantics)
- Internal layer 3: `npu_isa` dialect (near-ISA ops)

Note: MLIR dialect namespaces cannot contain dots, so the intended conceptual names
`npu.kernel`, `npu.schedule`, and `npu.isa` are represented as three dialects
`npu_kernel`, `npu_schedule`, and `npu_isa`.

## Current End-to-End Slice (Phase 1)

- Input: `linalg.matmul` on tensors
- Lowering:
  1. `linalg` -> `npu_kernel`
  2. `npu_kernel` -> `npu_schedule`
  3. `npu_schedule` -> `npu_isa`
- Output: simulator-compatible textual ISA emitted by `npu-translate`

The textual backend currently emits `dma.load`, `dma.load.mxu0`, `dma.wait`, and
`matmul.mxu0` forms compatible with the Python simulator ISA.

## LLVM/MLIR Pinning

Pin metadata is in:

- `third_party/llvm-project.pin.json`

The pin mirrors IREE release `v3.10.0` and its LLVM submodule SHA.

Bootstrap script:

```bash
./scripts/bootstrap_llvm.sh
```

## Toolchain Options

There are now 3 supported ways to provide LLVM/MLIR:

1. Any compatible prebuilt toolchain
   - set `LLVM_DIR` and `MLIR_DIR`
2. A Merlin build tree
   - set `MERLIN_BUILD_DIR`
3. A repo-local pinned LLVM checkout and build
   - use `./compiler/scripts/build_pinned_llvm.sh`

The most reproducible option for external users is the pinned repo-local build.

## Fresh Clone Reproduction

Run from repo root:

```bash
export LLVM_DIR=/path/to/llvm/lib/cmake/llvm
export MLIR_DIR=/path/to/mlir/lib/cmake/mlir
./compiler/scripts/configure_compiler.sh
cmake --build build/npu_compiler --target npu-opt npu-translate -j8
```

If you want to derive those from a Merlin build tree instead, set:

```bash
export MERLIN_BUILD_DIR=/path/to/merlin/build/host-vanilla-release
./compiler/scripts/configure_compiler.sh
```

The script only consumes CMake packages from the external toolchain. It does not
modify the external LLVM tree.

If you want a fully repo-local pinned toolchain instead:

```bash
./compiler/scripts/build_pinned_llvm.sh
export LLVM_DIR=$PWD/build/llvm-install/lib/cmake/llvm
export MLIR_DIR=$PWD/build/llvm-install/lib/cmake/mlir
./compiler/scripts/configure_compiler.sh
cmake --build build/npu_compiler --target npu-opt npu-translate -j8
```

## Reproduce FP8 Matmul Lowering

```bash
build/npu_compiler/tools/npu-opt \
  compiler/test/lowering/linalg_generic_fp8_ukernel_pipeline.mlir \
  -convert-linalg-to-npu-kernel \
  -convert-npu-kernel-to-schedule \
  -convert-npu-schedule-to-isa \
  > compiler/test/outputs/matmul_fp8_isa.mlir

build/npu_compiler/tools/npu-translate \
  --mlir-to-npu-text-isa compiler/test/outputs/matmul_fp8_isa.mlir \
  > compiler/test/outputs/matmul_fp8_isa.txt
```

## Save Full `--mlir-print-ir-after-all` Trace

```bash
build/npu_compiler/tools/npu-opt \
  compiler/test/lowering/linalg_generic_fp8_ukernel_pipeline.mlir \
  -convert-linalg-to-npu-kernel \
  -convert-npu-kernel-to-schedule \
  -convert-npu-schedule-to-isa \
  --mlir-disable-threading \
  --mlir-print-ir-after-all \
  --mlir-print-ir-module-scope \
  > compiler/test/outputs/matmul_fp8_print_after_all.txt 2>&1
```

Additional staged artifacts are generated by:

```bash
./compiler/scripts/show_ukernel_pipeline.sh \
  build/npu_compiler \
  compiler/test/lowering/linalg_generic_fp8_ukernel_pipeline.mlir \
  compiler/test/outputs/ukernel_pipeline
```

## Simulator Smoke Flow

Use emitted textual ISA with the local `model_npu` simulator:

```bash
uv run python compiler/scripts/run_simulator_smoke.py compiler/test/outputs/matmul_fp8_isa.txt
```

For Gemma-symbol ukernel smoke traces, generated outputs are available in:

- `compiler/test/outputs/gemma_mlp_isa.txt`
- `compiler/test/outputs/gemma_attention_isa.txt`

You can regenerate those with:

```bash
./compiler/scripts/emit_symbol_ukernel_isa.sh \
  build/npu_compiler \
  npu_uk_gemma_mlp_f8E4M3FN_f8E4M3FN_f32 \
  compiler/test/outputs/gemma_mlp_isa.txt

./compiler/scripts/emit_symbol_ukernel_isa.sh \
  build/npu_compiler \
  npu_uk_gemma_attention_f8E4M3FN_f8E4M3FN_f32 \
  compiler/test/outputs/gemma_attention_isa.txt
```

## Ukernel-Style Matmul Example (FP8)

An IREE-style `linalg.generic` matmul-to-ukernel flow is available in:

- `test/lowering/linalg_generic_fp8_ukernel_pipeline.mlir`

This lowers:

1. `linalg.generic` (matmul-like contraction, fp8/fp8->f32)
2. -> `npu_kernel.ukernel_generic`
3. -> `npu_schedule.ukernel_launch`
4. -> `npu_isa.*` skeleton
5. -> textual ISA

To inspect every IR stage plus full `--mlir-print-ir-after-all` output:

```bash
./scripts/show_ukernel_pipeline.sh ../build/npu_compiler
```
